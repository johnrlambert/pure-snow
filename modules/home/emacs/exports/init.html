1. Overview

1.1. Will the real init.org please stand up?

After years of vim/neovim use I finally took the plunge and decided to switch to emacs. This was something I agonized over. (Seriously. I am a loser.) I have years and years of experience and muscle memory with vim. Even though I probably took advantage of very few of its advanced features I still made my brain fit its paradigm.

What changed? Org-mode.

Org-mode allows code to exist coherently alongside text, notes and task management. In the age of LLMs this is particularly important as it creates a kind of shared artifact for me and my silicon buddies.

The most intriguing file to org-ify is the emacs configuration itself.

That means the real ~/.emacs.d/init.el is generated from this file during the Home Manager build. That gives me three nice properties at once:

  1. The configuration stays reproducible and Nix-friendly.
  2. The reasoning for each section can live right next to the code.
  3. The TODOs can live right alongside the code itself. That means that I can create something like:

1.1.1. TODO Write a sample todo :nix-config: :example-todo:

:ID: 01KT4816KR4EVKS5Y1CZANCFA7

and it will just transparently live with the code in situ but also be managed by globally by org-agenda.

2. Package bootstrap

This first block does the minimum work needed to talk to Emacs package archives and to ensure use-package is available.

One slightly frustrating thing is that emacs doesn't support this org and org-babel tangling business out of the box. So in order to make this somewhat intelligible to nix it has to lean on some preexisting customizations.

Other than that, it follows a pretty common approach:

  • MELPA is used for fast-moving community packages.
  • Org ELPA is added so Org can be updated independently when desired.
  • GNU ELPA stays available for the usual core add-ons.
  • package-initialize turns on installed packages.
  • If archive metadata is missing, Emacs refreshes package contents.
  • use-package becomes the common declaration form for the rest of the file.

Because this config is also deployed through Nix, this is a hybrid setup: Nix provides some packages declaratively, while Emacs package.el still pulls in others at runtime. That is a little "frankenstein", but it is a perfectly workable intermediate state.

BTW kids I remember before we had all these fancy language specific repositories. I spent half of 2006 looking for a .dll. 

;; Initialize package sources.
(require 'package)

(setq package-archives '(("melpa" . "https://melpa.org/packages/")
                         ("org" . "https://orgmode.org/elpa/")
                         ("elpa" . "https://elpa.gnu.org/packages/")))

(setq-default word-wrap t)

(package-initialize)

(unless package-archive-contents
  (package-refresh-contents))

;; Install use-package for easier package management.
(unless (package-installed-p 'use-package)
  (package-install 'use-package))

(require 'use-package)
(setq use-package-always-ensure t)

3. Theme

The theme choice is straightforward: load a Gruvbox variant and aggressively clear any previously enabled themes first.

My read of this is:

  • custom-safe-themes t is here to avoid repeated safe-theme prompts.
  • Disabling all current themes before loading the new one prevents theme stacking and weird face inheritance.
  • gruvbox-dark-medium is the preferred baseline.
(use-package gruvbox-theme
  :config
  (setq custom-safe-themes t)
  (mapc #'disable-theme custom-enabled-themes)
  (load-theme 'gruvbox-dark-medium t))

4. Typography, pitch, and an overly earnest Org art direction pass

This is where things get a little more editorial.

A lot of Emacs setups assume a single monospace font everywhere. That is a very reasonable default for programming, terminals, logs, and anything else where columns matter.

But Org mode is only sometimes code. A big part of my use of Org is closer to writing, outlining, note-making, and publishing than it is to editing source files. In those buffers, a tasteful proportional typeface can feel less like a terminal emulator wearing a fake moustache and more like an actual writing environment.

The package fixed-pitch-mode is interesting precisely because it flips the usual Emacs assumption:

  • the default face can be proportional,
  • code-oriented buffers can opt back into monospace automatically,
  • and Org can become a mixed-pitch environment where prose is pleasant but tables, code blocks, and metadata still line up.

That is the basic plan here.

For the actual type choices, I am going for a vaguely Bauhaus / modernist editorial vibe without turning Org into a design student crime scene:

  • Inter for body text and general UI readability,
  • Jost for headings, because it has a geometric, poster-ish, Futura-adjacent flavor that nudges things in a Bauhaus direction,
  • JetBrains Mono Nerd Font Mono for code, tables, src blocks, and all the places where character alignment still matters.

The key thing to keep in mind is that fixed-pitch-mode is not being used to make Org itself monospaced. Instead, it gives me a good global baseline for code-ish buffers, while Org gets a custom mixed-pitch treatment of its own.

5. Org mode foundation

This is the heart of the setup.

The overall intent seems to be:

  • make Org comfortable for everyday writing,
  • make source blocks pleasant to edit,
  • make HTML export look presentable out of the box, and
  • make links easier to follow, especially with Evil in the mix.

A few noteworthy choices:

  • org-startup-indented and hidden leading stars make files look cleaner.
  • org-ellipsis uses a custom fold marker.
  • org-return-follows-link makes RET more link-friendly.
  • a small GTD-ish TODO workflow gives tasks a little more structure than a single TODO/DONE pair.
  • logging completion / cancellation into drawers keeps historical metadata without cluttering the visible outline too badly.
  • source block settings make Babel editing behave more like a real code editor.
  • Org gets its own mixed-pitch typography pass so headings can feel a little more like a page layout and a little less like a man page.
  • svg-tag-mode is used as a small ornamental layer for TODO states and priorities because apparently regular Emacs was not yet Emacs enough.
  • the HTML export header uses a local stylesheet for a cleaner, self-contained export.

6. Indentation policy

This section standardizes indentation across a handful of languages.

The intent seems practical rather than ideological:

  • use 4 spaces for most programming languages,
  • use 2 spaces for CoffeeScript,
  • keep Evil shift width aligned with mode indentation so << and >> do what you expect.
(defun my/set-indent-4 ()
  "Use 4 spaces for indentation in the current buffer."
  (setq-local indent-tabs-mode nil)
  (setq-local tab-width 4)
  (setq-local standard-indent 4)
  (setq-local evil-shift-width 4))

(defun my/set-coffee-indent-2 ()
  "Use 2 spaces for indentation in CoffeeScript buffers."
  (setq-local indent-tabs-mode nil)
  (setq-local tab-width 2)
  (setq-local standard-indent 2)
  (setq-local coffee-tab-width 2)
  (setq-local evil-shift-width 2))

7. ULID support for Org headings

This is one of the more distinctive custom pieces.

The idea is to assign stable IDs to Org headings, but to do it with ULIDs instead of the more common random UUIDs or Org's own org-id helpers.

Why might this exist?

  • ULIDs are sortable by time.
  • They are shorter and friendlier than many UUID renderings.
  • Stable heading IDs make linking and refiling safer over time.

The code below:

  • defines a Crockford Base32 alphabet,
  • generates a time-sortable ULID-ish identifier,
  • adds an ID property to new headings,
  • stamps headings when pressing org-return, and
  • provides a whole-buffer repair command for backfilling missing IDs.

The note in the docstring is important: this is not strictly monotonic within the same millisecond, but for personal notes that is usually fine. 

(defconst my/ulid-alphabet "0123456789ABCDEFGHJKMNPQRSTVWXYZ"
  "Crockford Base32 alphabet used by ULIDs.")

(random t)

(defun my/ulid--encode-base32 (num len)
  "Encode NUM in Crockford Base32, left-padded to LEN characters."
  (let ((out (make-string len ?0)))
    (dotimes (i len out)
      (aset out (- len 1 i)
            (aref my/ulid-alphabet (% num 32)))
      (setq num (/ num 32)))))

(defun my/generate-ulid ()
  "Generate a ULID string.

This implementation is time-sortable, but not strictly monotonic when
multiple IDs are created within the same millisecond."
  (let* ((time-ms (floor (* 1000 (float-time))))
         (r1 (random (expt 2 20)))
         (r2 (random (expt 2 20)))
         (r3 (random (expt 2 20)))
         (r4 (random (expt 2 20)))
         (rand80 (+ (lsh r1 60)
                    (lsh r2 40)
                    (lsh r3 20)
                    r4)))
    (concat (my/ulid--encode-base32 time-ms 10)
            (my/ulid--encode-base32 rand80 16))))

(defun my/org-add-id-to-new-heading ()
  "Add a ULID-backed ID property to the current Org heading if missing."
  (when (derived-mode-p 'org-mode)
    (save-excursion
      (org-back-to-heading t)
      (unless (org-entry-get (point) "ID")
        (org-entry-put (point) "ID" (my/generate-ulid))))))

(defun my/org-add-id-at-point-if-heading ()
  "Add a ULID-backed ID to the heading at point, if there is one."
  (when (and (derived-mode-p 'org-mode)
             (org-at-heading-p))
    (my/org-add-id-to-new-heading)))

(defun my/org-return-stamp-id-before (&rest _)
  "Stamp the current Org heading with a ULID-backed ID before `org-return'."
  (my/org-add-id-at-point-if-heading))

(defun my/org-ulidize-buffer ()
  "Add ULID-backed ID properties to headings in the current Org buffer.

Existing IDs are preserved."
  (interactive)
  (unless (derived-mode-p 'org-mode)
    (user-error "Current buffer is not in Org mode"))
  (let ((count 0))
    (org-map-entries
     (lambda ()
       (unless (org-entry-get (point) "ID")
         (org-entry-put (point) "ID" (my/generate-ulid))
         (setq count (1+ count))))
     nil
     'file)
    (message "Added ULID IDs to %d heading%s"
             count
             (if (= count 1) "" "s"))))

8. Org export and Babel execution

This configuration treats Org as both a writing format and an executable notebook format.

The export helper writes HTML output into a local exports/ directory next to its source file and opens the result in a browser. That is a pleasant workflow for publishing one-off notes or drafts without having to configure a full site.

The Babel section enables a modest set of languages:

  • Emacs Lisp
  • shell
  • Python
  • Nix
  • JavaScript
  • CoffeeScript (really)
  • Mermaid for charting

Given the repository, the Nix support is especially relevant. 

(defun my/org-export-asset-files ()
  "Collect local file assets referenced by the current Org buffer."
  (let ((base-dir (file-name-directory (or (buffer-file-name) default-directory)))
        assets)
    (org-element-map (org-element-parse-buffer) 'link
      (lambda (link)
        (when (string= (org-element-property :type link) "file")
          (let* ((path (org-link-unescape (org-element-property :path link)))
                 (file (expand-file-name path base-dir)))
            (when (file-regular-p file)
              (push file assets))))))
    (org-element-map (org-element-parse-buffer) 'src-block
      (lambda (src)
        (let* ((params (org-babel-parse-header-arguments (or (org-element-property :parameters src) "")))
               (path (cdr (assq :file params)))
               (file (and path (expand-file-name path base-dir))))
          (when (and file (file-regular-p file))
            (push file assets)))))
    (delete-dups assets)))

(defun my/org-copy-export-assets (export-dir)
  "Copy local Org asset files into EXPORT-DIR, preserving relative paths when possible."
  (let ((base-dir (file-name-directory (or (buffer-file-name) default-directory))))
    (dolist (file (my/org-export-asset-files))
      (let* ((relative (file-relative-name file base-dir))
             (target (expand-file-name
                      (if (string-prefix-p "../" relative)
                          (file-name-nondirectory file)
                        relative)
                      export-dir)))
        (make-directory (file-name-directory target) t)
        (copy-file file target t)))
    (copy-file (locate-user-emacs-file "custom.css")
               (expand-file-name "custom.css" export-dir)
               t)))

(defun my/org-inline-export-css (backend)
  "Inline custom CSS for HTML BACKEND exports."
  (when (org-export-derived-backend-p backend 'html)
    (let ((css-file (locate-user-emacs-file "custom.css")))
      (setq-local org-html-head
                  (if (file-exists-p css-file)
                      (with-temp-buffer
                        (insert-file-contents css-file)
                        (format "<style>\n%s\n</style>\n" (buffer-string)))
                    "")))))

(defun my/org-export-ensure-support-files (orig backend file &rest args)
  "Export with ORIG, then ensure support files exist beside HTML output FILE."
  (let ((result (apply orig backend file args)))
    (when (eq backend 'html)
      (my/org-copy-export-assets (file-name-directory file)))
    result))

(defun my/org-render-mermaid-blocks ()
  "Render all Mermaid source blocks in the current Org buffer."
  (require 'ob-mermaid)
  (org-babel-map-src-blocks nil
    (let* ((info (org-babel-get-src-block-info 'light))
           (lang (car info))
           (params (nth 2 info))
           (file (cdr (assq :file params))))
      (when (string= lang "mermaid")
        (when file
          (make-directory (file-name-directory (expand-file-name file default-directory)) t))
        (org-babel-execute-src-block)))))

(defun my/org-export-to-html ()
  "Export the current Org buffer to HTML in an exports directory and open it."
  (interactive)
  (unless (derived-mode-p 'org-mode)
    (user-error "Current buffer is not in Org mode"))
  (let* ((source-file (or (buffer-file-name)
                          (expand-file-name "untitled.org" default-directory)))

9. Clipboard and rich paste

This section is an attempt to make Org feel a little more like a document editor without giving up plain text.

The desired behavior is roughly:

  • if the X clipboard contains an image, save it locally and insert an Org file link,
  • if the clipboard contains HTML, convert it into Org syntax before inserting,
  • otherwise fall back to an ordinary text paste.

That creates a smoother path from browsers, chat apps, and other GUI tools into an Org note.

The implementation below is intentionally modest and X11-specific:

  • it queries clipboard MIME targets with xclip,
  • saves pasted images into an images/ directory next to the current file,
  • uses pandoc to convert text/html clipboard data into Org markup,
  • falls back to yank when no richer path is available.

This is not the final word on rich paste, but it is a good foundation. 

(defun my/clipboard-targets ()
  "Return available X clipboard targets as a list of MIME types."
  (when (executable-find "xclip")
    (split-string
     (shell-command-to-string
      "xclip -selection clipboard -t TARGETS -o 2>/dev/null || true")
     "\n" t)))

(defun my/clipboard-has-target-p (target)
  "Return non-nil when the X clipboard advertises TARGET."
  (member target (my/clipboard-targets)))

(defun my/org-paste-image-from-clipboard ()
  "Save a PNG image from the clipboard and insert an Org file link."
  (interactive)
  (unless (my/clipboard-has-target-p "image/png")
    (user-error "Clipboard does not contain image/png"))
  (let* ((base-dir (file-name-directory (or (buffer-file-name) default-directory)))
         (image-dir (expand-file-name "images" base-dir))
         (filename (format-time-string "paste-%Y%m%d-%H%M%S.png"))
         (full-path (expand-file-name filename image-dir))
         (relative-path (file-relative-name full-path base-dir))
         (command (format "xclip -selection clipboard -t image/png -o > %s"
                          (shell-quote-argument full-path))))
    (make-directory image-dir t)
    (unless (zerop (shell-command command))
      (user-error "Failed to save image from clipboard"))
    (insert (format "[[file:%s]]" relative-path))
    (when (derived-mode-p 'org-mode)
      (org-display-inline-images))
    full-path))

(defun my/org-paste-html-as-org ()
  "Convert HTML clipboard contents to Org and insert the result."
  (interactive)
  (unless (executable-find "pandoc")
    (user-error "pandoc is not available"))
  (unless (my/clipboard-has-target-p "text/html")
    (user-error "Clipboard does not contain text/html"))
  (let* ((command "xclip -selection clipboard -t text/html -o | pandoc --wrap=none -f html -t org")
         (org-text (shell-command-to-string command)))
    (insert (replace-regexp-in-string "\n\\'" "" org-text))))

(defun my/org-paste-dwim ()
  "Paste clipboard content into Org, preferring images and HTML when available."
  (interactive)
  (cond
   ((not (derived-mode-p 'org-mode))
    (yank))
   ((my/clipboard-has-target-p "image/png")
    (my/org-paste-image-from-clipboard))
   ((and (my/clipboard-has-target-p "text/html")
         (executable-find "pandoc"))
    (my/org-paste-html-as-org))
   (t
    (yank))))

(with-eval-after-load 'org
  (define-key org-mode-map (kbd "C-c y") #'my/org-paste-dwim))

18. Evil mode

This is the modal editing section.

The configuration does a few careful things here:

  • ensures Evil is installed even in a package.el world,
  • tells Evil to use Emacs' built-in undo/redo backend,
  • enables Evil globally,
  • reinforces C-h/j/k/l window movement inside Evil states,
  • gives Org buffers sane normal-state bindings for folding and link opening,
  • makes special-purpose buffers start in the most natural state.

That last part matters. Chat views, agenda screens, transient popups, and terminal-like prompts all want different default modes if Evil is enabled. The hooks below encode those preferences.
