* Package bootstrap

This first block does the minimum work needed to talk to Emacs package
archives and to ensure =use-package= is available.

One slightly frustrating thing is that emacs doesn't support this org and org-babel tangling business out of the box. So in order to make this somewhat intelligible to nix it has to lean on some preexisting customizations.

Other than that, it follows a pretty common approach:

- MELPA is used for fast-moving community packages.
- Org ELPA is added so Org can be updated independently when desired.
- GNU ELPA stays available for the usual core add-ons.
- =package-initialize= turns on installed packages.
- If archive metadata is missing, Emacs refreshes package contents.
- =use-package= becomes the common declaration form for the rest of the file.

Because this config is also deployed through Nix, this is a hybrid setup:
Nix provides some packages declaratively, while Emacs package.el still pulls
in others at runtime. That is a little "frankenstein", but it is a perfectly
workable intermediate state.

BTW kids I remember before we had all these fancy language specific repositories. I spent half of 2006 looking for a .dll. 

#+begin_src emacs-lisp
;; Initialize package sources.
(require 'package)

(setq package-archives '(("melpa" . "https://melpa.org/packages/")
                         ("org" . "https://orgmode.org/elpa/")
                         ("elpa" . "https://elpa.gnu.org/packages/")))

(setq-default word-wrap t)

(package-initialize)

(unless package-archive-contents
  (package-refresh-contents))

;; Install use-package for easier package management.
(unless (package-installed-p 'use-package)
  (package-install 'use-package))

(require 'use-package)
(setq use-package-always-ensure t)
#+end_src

* Theme

The theme choice is straightforward: load a Gruvbox variant and aggressively
clear any previously enabled themes first.

My read of this is:

- =custom-safe-themes t= is here to avoid repeated safe-theme prompts.
- Disabling all current themes before loading the new one prevents theme
  stacking and weird face inheritance.
- =gruvbox-dark-medium= is the preferred baseline.

#+begin_src emacs-lisp
(use-package gruvbox-theme
  :config
  (setq custom-safe-themes t)
  (mapc #'disable-theme custom-enabled-themes)
  (load-theme 'gruvbox-dark-medium t))
#+end_src
* Typography, pitch, and an overly earnest Org art direction pass

This is where things get a little more editorial.

A lot of Emacs setups assume a single monospace font everywhere. That is a
very reasonable default for programming, terminals, logs, and anything else
where columns matter.

But Org mode is only *sometimes* code. A big part of my use of Org is closer
to writing, outlining, note-making, and publishing than it is to editing
source files. In those buffers, a tasteful proportional typeface can feel less
like a terminal emulator wearing a fake moustache and more like an actual
writing environment.

The package =fixed-pitch-mode= is interesting precisely because it flips the
usual Emacs assumption:

- the *default* face can be proportional,
- code-oriented buffers can opt back into monospace automatically,
- and Org can become a mixed-pitch environment where prose is pleasant but
  tables, code blocks, and metadata still line up.

That is the basic plan here.

For the actual type choices, I am going for a vaguely Bauhaus / modernist
editorial vibe without turning Org into a design student crime scene:

- **Inter** for body text and general UI readability,
- **Jost** for headings, because it has a geometric, poster-ish, Futura-adjacent
  flavor that nudges things in a Bauhaus direction,
- **JetBrains Mono Nerd Font Mono** for code, tables, src blocks, and all the
  places where character alignment still matters.

The key thing to keep in mind is that =fixed-pitch-mode= is *not* being used to
make Org itself monospaced. Instead, it gives me a good global baseline for
code-ish buffers, while Org gets a custom mixed-pitch treatment of its own.

#+begin_src emacs-lisp
(defconst my/font-body "Inter"
  "Primary proportional typeface for prose and general UI text.")

(defconst my/font-headings "Jost"
  "Geometric sans face used for Org headings and titles.")

(defconst my/font-mono "JetBrainsMono Nerd Font Mono"
  "Monospace face used where character alignment matters.")

(add-to-list 'load-path (expand-file-name "site-lisp" user-emacs-directory))

(defun my/apply-base-typography ()
  "Apply the global font baseline used across frames."
  (setq default-frame-alist
        (assq-delete-all 'font default-frame-alist))
  (add-to-list 'default-frame-alist `(font . ,(format "%s-12" my/font-body)))
  (set-face-attribute 'default nil :family my/font-body :height 115 :weight 'regular)
  (set-face-attribute 'variable-pitch nil :family my/font-body :height 1.0 :weight 'regular)
  (set-face-attribute 'fixed-pitch nil :family my/font-mono :height 105 :weight 'regular)
  (set-face-attribute 'mode-line nil :family my/font-body :height 0.95)
  (set-face-attribute 'mode-line-inactive nil :family my/font-body :height 0.95)
  (set-face-attribute 'header-line nil :family my/font-body :height 0.95)
  (set-face-attribute 'tab-bar nil :family my/font-body :height 0.95)
  (set-face-attribute 'tab-line nil :family my/font-body :height 0.95)
  (set-face-attribute 'minibuffer-prompt nil :family my/font-body :weight 'semibold))

(my/apply-base-typography)

(use-package fixed-pitch
  :ensure nil
  :init
  ;; The package installs its hooks at load time, so the customization that
  ;; decides *which* buffers become monospace needs to be in place before the
  ;; package itself is required.
  (setq fixed-pitch-dont-change-cursor t)
  (setq fixed-pitch-use-extended-default t)
  (setq fixed-pitch-whitelist-hooks
        '(conf-mode-hook
          yaml-mode-hook))
  (setq fixed-pitch-blacklist-hooks
        '(org-mode-hook))
  :config
  ;; `fixed-pitch-mode' gives prog-mode, comint, dired, Magit, and friends a
  ;; reliable monospaced baseline while letting the rest of Emacs breathe a
  ;; bit more typographically.
  (message "fixed-pitch-mode loaded: code buffers stay monospace, prose can relax"))
#+end_src

* Org mode foundation

This is the heart of the setup.

The overall intent seems to be:

- make Org comfortable for everyday writing,
- make source blocks pleasant to edit,
- make HTML export look presentable out of the box, and
- make links easier to follow, especially with Evil in the mix.

A few noteworthy choices:

- =org-startup-indented= and hidden leading stars make files look cleaner.
- =org-ellipsis= uses a custom fold marker.
- =org-return-follows-link= makes =RET= more link-friendly.
- a small GTD-ish TODO workflow gives tasks a little more structure than a
  single =TODO/DONE= pair.
- logging completion / cancellation into drawers keeps historical metadata
  without cluttering the visible outline too badly.
- source block settings make Babel editing behave more like a real code editor.
- Org gets its own mixed-pitch typography pass so headings can feel a little
  more like a page layout and a little less like a man page.
- =svg-tag-mode= is used as a small ornamental layer for TODO states and
  priorities because apparently regular Emacs was not yet Emacs enough.
- the HTML export header uses a local stylesheet for a cleaner, self-contained export.

  (defun my/org-find-metadata-drawer ()
    "Move point to the Metadata heading and return non-nil when found."
    (goto-char (point-min))
    (re-search-forward "^\\* Metadata\\b" nil t))

  (defun my/org-metadata-value (key)
    "Return property KEY from the top-level Metadata heading."
    (save-excursion
      (when (my/org-find-metadata-drawer)
        (let ((value (org-entry-get nil key)))
          (when value
            (setq value
                  (replace-regexp-in-string
                   "{{{today}}}"
                   (format-time-string "%B %d, %Y")
                   value t t)))
          value))))

  (defun my/org-latex-escape (text)
    "Escape a small set of LaTeX-special characters in TEXT."
    (let ((value (or text "")))
      (dolist (pair '(("\\" . "\\textbackslash{}")
                      ("&" . "\\&")
                      ("%" . "\\%")
                      ("$" . "\\$")
                      ("#" . "\\#")
                      ("_" . "\\_")
                      ("{" . "\\{")
                      ("}" . "\\}")))
        (setq value (replace-regexp-in-string
                     (regexp-quote (car pair))
                     (cdr pair)
                     value t t)))
      value))

* ULID support for Org headings

This is one of the more distinctive custom pieces.

The idea is to assign stable IDs to Org headings, but to do it with ULIDs
instead of the more common random UUIDs or Org's own =org-id= helpers.

The code below:

- defines a Crockford Base32 alphabet,
- generates a time-sortable ULID-ish identifier,
- adds an =ID= property to new headings,
- stamps headings when pressing =org-return=, and
- provides a whole-buffer repair command for backfilling missing IDs.

The note in the docstring is important: this is *not strictly monotonic*
within the same millisecond, but for personal notes that is usually fine.

#+begin_src emacs-lisp
(defconst my/ulid-alphabet "0123456789ABCDEFGHJKMNPQRSTVWXYZ"
  "Crockford Base32 alphabet used by ULIDs.")

(random t)

(defun my/ulid--encode-base32 (num len)
  "Encode NUM in Crockford Base32, left-padded to LEN characters."
  (let ((out (make-string len ?0)))
    (dotimes (i len out)
      (aset out (- len 1 i)
            (aref my/ulid-alphabet (% num 32)))
      (setq num (/ num 32)))))

(defun my/generate-ulid ()
  "Generate a ULID string.

This implementation is time-sortable, but not strictly monotonic when
multiple IDs are created within the same millisecond."
  (let* ((time-ms (floor (* 1000 (float-time))))
         (r1 (random (expt 2 20)))
         (r2 (random (expt 2 20)))
         (r3 (random (expt 2 20)))
         (r4 (random (expt 2 20)))
         (rand80 (+ (lsh r1 60)
                    (lsh r2 40)
                    (lsh r3 20)
                    r4)))
    (concat (my/ulid--encode-base32 time-ms 10)
            (my/ulid--encode-base32 rand80 16))))

(defun my/org-add-id-to-new-heading ()
  "Add a ULID-backed ID property to the current Org heading if missing."
  (when (derived-mode-p 'org-mode)
    (save-excursion
      (org-back-to-heading t)
      (unless (org-entry-get (point) "ID")
        (org-entry-put (point) "ID" (my/generate-ulid))))))

(defun my/org-add-id-at-point-if-heading ()
  "Add a ULID-backed ID to the heading at point, if there is one."
  (when (and (derived-mode-p 'org-mode)
             (org-at-heading-p))
    (my/org-add-id-to-new-heading)))

(defun my/org-return-stamp-id-before (&rest _)
  "Stamp the current Org heading with a ULID-backed ID before `org-return'."
  (my/org-add-id-at-point-if-heading))

(defun my/org-ulidize-buffer ()
  "Add ULID-backed ID properties to headings in the current Org buffer.

Existing IDs are preserved."
  (interactive)
  (unless (derived-mode-p 'org-mode)
    (user-error "Current buffer is not in Org mode"))
  (let ((count 0))
    (org-map-entries
     (lambda ()
       (unless (org-entry-get (point) "ID")
         (org-entry-put (point) "ID" (my/generate-ulid))
         (setq count (1+ count))))
     nil
     'file)
    (message "Added ULID IDs to %d heading%s"
             count
             (if (= count 1) "" "s"))))
#+end_src

(defun my/org-export-asset-files ()
  "Collect local file assets referenced by the current Org buffer."
  (let ((base-dir (file-name-directory (or (buffer-file-name) default-directory)))
        assets)
    (org-element-map (org-element-parse-buffer) 'link
      (lambda (link)
        (when (string= (org-element-property :type link) "file")
          (let* ((path (org-link-unescape (org-element-property :path link)))
                 (file (expand-file-name path base-dir)))
            (when (file-regular-p file)
              (push file assets))))))
    (org-element-map (org-element-parse-buffer) 'src-block
      (lambda (src)
        (let* ((params (org-babel-parse-header-arguments (or (org-element-property :parameters src) "")))
               (path (cdr (assq :file params)))
               (file (and path (expand-file-name path base-dir))))
          (when (and file (file-regular-p file))
            (push file assets)))))

* Clipboard and rich paste

This section is an attempt to make Org feel a little more like a document
editor without giving up plain text.

The desired behavior is roughly:

- if the X clipboard contains an image, save it locally and insert an Org file
  link,
- if the clipboard contains HTML, convert it into Org syntax before inserting,
- otherwise fall back to an ordinary text paste.

That creates a smoother path from browsers, chat apps, and other GUI tools into
an Org note.

The implementation below is intentionally modest and X11-specific:

- it queries clipboard MIME targets with =xclip=,
- saves pasted images into an =images/= directory next to the current file,
- uses =pandoc= to convert =text/html= clipboard data into Org markup,
- falls back to =yank= when no richer path is available.

This is not the final word on rich paste, but it is a good foundation.

#+begin_src emacs-lisp
(defun my/clipboard-targets ()
  "Return available X clipboard targets as a list of MIME types."
  (when (executable-find "xclip")
    (split-string
     (shell-command-to-string
      "xclip -selection clipboard -t TARGETS -o 2>/dev/null || true")
     "\n" t)))

(defun my/clipboard-has-target-p (target)
  "Return non-nil when the X clipboard advertises TARGET."
  (member target (my/clipboard-targets)))

(defun my/org-paste-image-from-clipboard ()
  "Save a PNG image from the clipboard and insert an Org file link."
  (interactive)
  (unless (my/clipboard-has-target-p "image/png")
    (user-error "Clipboard does not contain image/png"))
  (let* ((base-dir (file-name-directory (or (buffer-file-name) default-directory)))
         (image-dir (expand-file-name "images" base-dir))
         (filename (format-time-string "paste-%Y%m%d-%H%M%S.png"))
         (full-path (expand-file-name filename image-dir))
         (relative-path (file-relative-name full-path base-dir))
         (command (format "xclip -selection clipboard -t image/png -o > %s"
                          (shell-quote-argument full-path))))
    (make-directory image-dir t)
    (unless (zerop (shell-command command))
      (user-error "Failed to save image from clipboard"))
    (insert (format "[[file:%s]]" relative-path))
    (when (derived-mode-p 'org-mode)
      (org-display-inline-images))
    full-path))

(defun my/org-paste-html-as-org ()
  "Convert HTML clipboard contents to Org and insert the result."
  (interactive)
  (unless (executable-find "pandoc")
    (user-error "pandoc is not available"))
  (unless (my/clipboard-has-target-p "text/html")
    (user-error "Clipboard does not contain text/html"))
  (let* ((command "xclip -selection clipboard -t text/html -o | pandoc --wrap=none -f html -t org")
         (org-text (shell-command-to-string command)))
    (insert (replace-regexp-in-string "\n\\'" "" org-text))))

(defun my/org-paste-dwim ()
  "Paste clipboard content into Org, preferring images and HTML when available."
  (interactive)
  (cond
   ((not (derived-mode-p 'org-mode))
    (yank))
   ((my/clipboard-has-target-p "image/png")
    (my/org-paste-image-from-clipboard))
   ((and (my/clipboard-has-target-p "text/html")
         (executable-find "pandoc"))
    (my/org-paste-html-as-org))
   (t
    (yank))))

(with-eval-after-load 'org
  (define-key org-mode-map (kbd "C-c y") #'my/org-paste-dwim))
#+end_src

- capture new tasks into =~/chronofile/inbox.org=.

This is a nice match for an Org-first workflow without requiring a large extra
framework such as Org-roam.

  (expand-file-name "memos/" org-directory)

  "Return the absolute path for capture template snippet NAME."
  (expand-file-name name my/org-capture-template-directory))

(defun my/org-slugify (text)
  "Convert TEXT into a lowercase filesystem-friendly slug."
  (let* ((downcased (downcase (string-trim (or text ""))))
         (slug (replace-regexp-in-string "[^a-z0-9]+" "-" downcased)))
    (string-trim slug "-+" "-+")))

(defun my/org-capture-memo-target ()
  "Create or visit a new memo file for capture."
  (let* ((subject (read-string "Memo subject: "))
         (recipient (read-string "To: "))
         (slug (my/org-slugify subject))
         (timestamp (format-time-string "%Y%m%d-%H%M%S"))
         (file (expand-file-name
                (format "%s-%s.org" timestamp (if (string-empty-p slug) "memo" slug))
                my/org-memo-directory)))
    (org-capture-put :memo-subject subject
                     :memo-to recipient
                     :memo-date (format-time-string "%B %d, %Y")
                     :memo-file file)
    (make-directory my/org-memo-directory t)
    (set-buffer (org-capture-target-buffer file))
    (widen)
    (goto-char (point-min))))

(defun my/org-capture-memo-template ()
  "Return a full-file memo template populated from capture prompts."
  (let* ((template (org-file-contents (my/org-capture-template-file "memo.org")))
         (subject (or (org-capture-get :memo-subject) "Memo"))
         (recipient (or (org-capture-get :memo-to) ""))
         (date (or (org-capture-get :memo-date) (format-time-string "%B %d, %Y")))
         (title (if (string-empty-p subject) "Memo" subject)))
    (setq template (replace-regexp-in-string "__TITLE__" title template t t))
    (setq template (replace-regexp-in-string "__TO__" recipient template t t))
    (setq template (replace-regexp-in-string "__DATE__" date template t t))
    (setq template (replace-regexp-in-string "__SUBJECT__" subject template t t))
    template))

(defun my/org-chronofile-files ()
  "Return usable Org files under `org-directory'."
  (seq-filter
   (lambda (path)
     (let ((base (file-name-nondirectory path)))
       (and (file-regular-p path)
            (not (string-prefix-p ".#" base))
            (not (and (string-prefix-p "#" base)
                      (string-suffix-p "#" base)))
            (not (string-match-p "/node_modules/" path)))))
   (directory-files-recursively (expand-file-name org-directory) "\\.org\\'")))

(defun my/org-refresh-agenda-files ()
  "Refresh `org-agenda-files' from `org-directory'."
  (setq org-agenda-files (my/org-chronofile-files)))

(defun my/org-agenda ()
  "Open `org-agenda' using all Org files in `~/chronofile'."
  (interactive)
  (my/org-refresh-agenda-files)
  (call-interactively #'org-agenda))

(defun my/org-chronofile-todos ()
  "Show all TODO items from Org files under `~/chronofile'."
  (interactive)
  (my/org-refresh-agenda-files)
  (org-todo-list))

* Evil mode

This is the modal editing section.

The configuration does a few careful things here:

- ensures Evil is installed even in a package.el world,
- tells Evil to use Emacs' built-in undo/redo backend,
- enables Evil globally,
- reinforces =C-h/j/k/l= window movement inside Evil states,
- gives Org buffers sane normal-state bindings for folding and link opening,
- makes special-purpose buffers start in the most natural state.

That last part matters. Chat views, agenda screens, transient popups, and
terminal-like prompts all want different default modes if Evil is enabled.
The hooks below encode those preferences.

#+begin_src emacs-lisp
(unless (package-installed-p 'evil)
  (package-install 'evil))

(require 'evil)
(setq evil-undo-system 'undo-redo)
(evil-mode 1)

(dolist (state '(normal motion visual))
  (evil-global-set-key state (kbd "C-h") #'windmove-left)
  (evil-global-set-key state (kbd "C-j") #'windmove-down)
  (evil-global-set-key state (kbd "C-k") #'windmove-up)
  (evil-global-set-key state (kbd "C-l") #'windmove-right))

(with-eval-after-load 'org
  (dolist (state '(normal motion))
    (evil-define-key state org-mode-map (kbd "TAB") #'org-cycle)
    (evil-define-key state org-mode-map (kbd "<tab>") #'org-cycle)
    (evil-define-key state org-mode-map (kbd "S-TAB") #'org-shifttab)
    (evil-define-key state org-mode-map (kbd "<backtab>") #'org-shifttab)
    (evil-define-key state org-mode-map (kbd "RET") #'org-open-at-point)))

(with-eval-after-load 'evil
  (evil-ex-define-cmd "org-agenda" #'my/org-agenda)
  (evil-ex-define-cmd "todos" #'my/org-chronofile-todos)
  (evil-ex-define-cmd "org-todos" #'my/org-chronofile-todos))

(with-eval-after-load 'pi-coding-agent
  (evil-set-initial-state 'pi-coding-agent-chat-mode 'normal)
  (evil-set-initial-state 'pi-coding-agent-input-mode 'insert)
  (add-hook 'pi-coding-agent-chat-mode-hook #'evil-normal-state)
  (add-hook 'pi-coding-agent-input-mode-hook #'evil-insert-state))
